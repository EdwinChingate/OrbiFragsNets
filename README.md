## What is OrbiFragsNets?

OrbiFragsNets is an **offline, database-independent Python tool** for the 
automatic *de novo* annotation of data-dependent MS2 (DDMS2) spectra 
generated by Orbitrap instruments.

Unlike conventional annotation tools, OrbiFragsNets does **not** rely on 
spectral libraries, machine learning models, or external databases. 
Instead, it derives annotations purely from:

1. The **statistical uncertainty** of every individual peak in the raw 
   Orbitrap profile data.
2. The **chemical consistency** of candidate fragment annotations — 
   expressed as a network of valid decomposition reactions.

The annotation process begins before centroiding, recovering the 
peak-specific confidence interval that is discarded in all conventional 
workflows. These confidence intervals replace the user-defined fixed mass 
error (ppm) found in other tools, making the annotation statistically 
justified rather than arbitrary.

OrbiFragsNets was benchmarked against RMassBank and SIRIUS on publicly 
available spectra for sulfamethoxazole, carbamazepine, and atenolol, 
producing equivalent or superior annotation results.

## How does it work?

The annotation pipeline has four conceptual stages, illustrated in the 
figure below.

```
MS1 profile data (raw .mzML)
        │
        ▼
┌───────────────────┐
│  Feature Detection│  ← Chromatographic peak extraction,
│  (FeaturesDet)    │    baseline correction, statistical
└────────┬──────────┘    characterization per feature.
         │
         ▼
┌───────────────────┐
│  MS2 Extraction   │  ← Peak-specific confidence interval
│  (MS2Spectrum)    │    derived from raw profile data.
└────────┬──────────┘
         │
         ▼
┌───────────────────────────┐
│  Formula Space Generation │  ← All chemically feasible formulas
│  (FragSpacePos,           │    per peak, filtered by statistical
│   MoleculesCand,          │    confidence interval.
│   SolveSpace)             │
└────────┬──────────────────┘
         │
         ▼
┌─────────────────────────────────┐
│  Fragments Networks             │  ← Annotations as graph nodes.
│  (SelfConsistFrag, AllNet,      │    Chemical consistency as edges.
│   GradeNet, AnnotateSpec)       │    Most connected network wins.
└─────────────────────────────────┘
         │
         ▼
  Annotated MS2 spectrum
  (molecular formulas, predicted
   m/z, mass error, confidence
   interval, relative intensity)
```

### Stage 1 — Feature Detection

The precursor ion is located in the MS1 data across retention time. 
Multiple overlapping chromatographic peaks are resolved by derivative 
analysis and valley detection. Baseline correction and statistical 
summarization yield a feature-level confidence interval for the precursor 
m/z.

### Stage 2 — MS2 Spectrum Extraction

The raw profile-mode MS2 scan is extracted and processed peak by peak. 
Each peak receives its own statistical confidence interval, computed from 
the intensity-weighted variance of its m/z distribution — the 
Orbitrap-specific peak shape that centroiding discards.

### Stage 3 — Formula Space Generation

For each MS2 peak, all chemically feasible molecular formulas are 
enumerated within the bounds set by the precursor formula and the 
peak-specific confidence interval. This is the chemical space of 
possible annotations for each fragment.

### Stage 4 — Fragments Networks and Selection

All candidate annotation combinations are expressed as **fragments 
networks** — graphs where nodes are candidate formula annotations and 
edges connect pairs of annotations whose mass difference corresponds to a 
valid neutral molecule (chemical consistency). The network with the 
highest number of consistent edges is selected as the annotation for the 
entire MS2 spectrum.

## Key Concepts

### Peak-Specific Confidence Interval

In Orbitrap instruments, the m/z for each ion is derived from the 
distribution of image-current frequencies measured in the ion trap. This 
distribution contains statistical information that is lost during 
centroiding. OrbiFragsNets recovers this information from the raw profile 
data, computing for every peak:

- Intensity-weighted mean m/z
- Intensity-weighted variance
- Student t-based confidence interval in ppm
- Shapiro-Wilk normality test

The result is a **peak-specific confidence interval** — replacing the 
global user-defined mass error used by all conventional tools.

### Chemical Consistency

Fragmentation is modelled as a decomposition reaction:

```
A⁺  →  B⁺  +  C
```

Two fragment annotations are **chemically consistent** if:

1. Their elemental difference is non-negative (stoichiometrically valid).
2. The exact mass of the difference matches the measured mass difference 
   within the combined confidence interval of both peaks.

Consistency is encoded as edges in the fragments network.

### Fragments Networks

A fragments network is a graph where:

- **Nodes** = one candidate formula annotation per MS2 peak
- **Edges** = pairwise chemical consistency between annotations

The **grade** (total edge count) of a network measures its overall 
chemical coherence. The annotation that maximizes the grade is selected.

This is equivalent to finding the maximum internally consistent 
annotation assignment — a constrained combinatorial optimization over 
the chemical space of all candidate formulas.

## Installation

### Requirements

- Linux OS (recommended) or any system supporting Python 3 and Jupyter
- Python 3.7 or higher
- ProteoWizard msConvert (for raw data conversion)

### Steps

1. Clone or download the repository:

```bash
git clone https://github.com/EdwinChingate/OrbiFragsNets.git
cd OrbiFragsNets
```

Or download the ZIP from GitHub and extract it.

2. Verify the following structure exists:

```
OrbiFragsNets/
├── OrbiFragsNets_Notebook.ipynb
├── Data/
├── Functions/
└── Parameters/
    ├── MassVec.csv
    ├── MaxAtomicSubscripts.csv
    └── ParametersTable.csv
```

3. Install dependencies (see below).

## Usage

### 1. Convert raw data to .mzML

Use ProteoWizard msConvert to convert your raw instrument file. 
**Keep data in profile mode. Do not apply any filters.**

```bash
msconvert "$file" --mzML
```

Move the resulting `.mzML` file to the `Data/` folder.

### 2. Configure parameters

Edit `Parameters/MaxAtomicSubscripts.csv` to set the maximum number of 
atoms expected per element. Use large values (>50) if the molecule is 
unknown, accepting longer execution times.

Edit `Parameters/ParametersTable.csv` to set processing parameters. 
The most critical is `NoiseTresInt` — the intensity threshold for noise 
filtering, which strongly affects execution time.

### 3. Open the Jupyter notebook

```bash
jupyter notebook OrbiFragsNets_Notebook.ipynb
```

### 4. Load your data

```python
DataSetName = 'your_file.mzML'
DataSet = ChargeDataSet(DataSetName)
```

### 5. Run the annotation

Provide the precursor m/z as the protonated adduct [M+H]⁺:

```python
PrecursorFragmentMass = 254.0593  # example: sulfamethoxazole [M+H]+
Annotation = OrbiFragsNets(PrecursorFragmentMass, DataSet)
```

For negative ionization mode, change the `ionization` parameter in 
`ParametersTable.csv` from `+` to `-`.

### 6. View and export results

```python
ShowDF(Annotation)
Annotation.to_csv('annotation_result.csv')
Annotation.to_excel('annotation_result.xlsx')
```

## Input and Output

### Input

| Item                  | Format    | Description                                   |
|-----------------------|-----------|-----------------------------------------------|
| MS data file          | `.mzML`   | Profile-mode Orbitrap LC-MS/MS data           |
| Precursor mass        | `float`   | Exact m/z of [M+H]⁺ or [M-H]⁻ precursor ion  |
| MaxAtomicSubscripts   | `.csv`    | Maximum atom counts per element                |
| ParametersTable       | `.csv`    | Processing and filtering parameters           |
| MassVec               | `.csv`    | Exact monoisotopic masses per element         |

### Output

A pandas DataFrame with one row per annotated fragment containing:

| Column                  | Description                                      |
|-------------------------|--------------------------------------------------|
| `Formula`               | Annotated molecular formula (e.g. C6H5NO2+)     |
| `MeassuredMZ`           | Measured centroid m/z (Da)                       |
| `PredictedMZ`           | Exact mass of annotated formula (Da)             |
| `Error`                 | Mass error in ppm                                |
| `ConfidenceInterval`    | Peak-specific confidence interval (ppm)          |
| `RelativeIntensity`     | Relative intensity (%)                           |
| `NumberofDataPoints`    | Raw data points clustered in this peak           |
| `K,Na,C13,C,Cl,...`     | Atomic subscripts per element                    |

## Parameters

### ParametersTable.csv

| Parameter                  | Description                                              |
|----------------------------|----------------------------------------------------------|
| `NoiseTresInt`             | Minimum intensity for a signal to be considered          |
| `MinInttobePeak`           | Minimum total intensity for a peak to be retained        |
| `ConfidenceIntervalTolerance` | Maximum allowed confidence interval in ppm            |
| `MinRelIntCont`            | Minimum relative intensity contribution (%)              |
| `MinSignalstobePeak`       | Minimum number of raw signals to form a peak             |
| `MassError`                | Fixed mass error for MS2 matching (fallback, ppm)        |
| `RTError`                  | Retention time tolerance for MS2 matching (seconds)      |
| `MinIntExplained`          | Minimum % of total intensity explained by a network      |
| `ionization`               | Ionization mode: `+` (positive) or `-` (negative)        |

### MaxAtomicSubscripts.csv

Defines the upper bound for each element in the formula search:

| Element | Recommended for unknowns |
|---------|--------------------------|
| K       | 1                        |
| Na      | 1                        |
| C13     | 2                        |
| C       | 50                       |
| Cl      | 3                        |
| S34     | 2                        |
| S       | 5                        |
| P       | 3                        |
| F       | 10                       |
| O       | 20                       |
| N       | 10                       |
| H       | 100                      |

## Performance and Benchmarks

OrbiFragsNets was tested on three pharmaceutical compounds from the 
MassBank database:

| Compound         | Fragments | Match with SIRIUS | Match with RMassBank |
|------------------|-----------|-------------------|----------------------|
| Sulfamethoxazole | 16        | 14/16             | 14/16                |
| Carbamazepine    | 3         | 2/3               | 2/3                  |
| Atenolol         | 18        | 16/18             | 16/18                |

Fragments annotated exclusively by RMassBank were found to be chemically 
inconsistent with the rest of the spectrum by both OrbiFragsNets and 
SIRIUS, suggesting they represent noise or artefacts rather than true 
fragments.

### Execution time

Execution time depends primarily on:

- Number of peaks in the MS2 spectrum
- Values in `MaxAtomicSubscripts.csv`
- Value of `NoiseTresInt`

The combinatorial search is exponential in the worst case. For spectra 
with more than ~20 peaks and large atomic bounds, execution may take 
several minutes. Reducing `MaxAtomicSubscripts` or increasing 
`NoiseTresInt` will significantly reduce execution time.

## Limitations

- **Formula-level only**: OrbiFragsNets annotates molecular formulas, 
  not molecular structures. It does not distinguish structural isomers.

- **Single charge only**: Multiply charged fragment ions are not 
  currently supported.

- **Profile-mode required**: The peak-specific confidence interval 
  calculation requires raw profile-mode data. Centroided input data 
  falls back to a fixed 10 ppm tolerance.

- **Computational scaling**: The brute-force combinatorial search 
  scales exponentially with the number of candidate formulas per peak. 
  Large molecules with many peaks and broad atomic bounds will require 
  significant computation time.

- **Positive and negative ionization**: Both modes are supported via the 
  `ionization` parameter, but adduct handling is currently limited to 
  [M+H]⁺ and [M-H]⁻.

- **Elements considered**: K, Na, ¹³C, C, Cl, ³⁴S, S, P, F, O, N, H. 
  Other elements are not currently supported.
